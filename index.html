    Seção de alertas e recomendações

    > JavaScript é Case Sensitive.

    > Inicializar as variáveis com valor indefinido.

    > Não instancie booleans com 'new Boolean(...)', pois isso é um objeto, o que é mais lento(execução) e não pode ser comparado com == ou ===. Instancie com false ou true mesmo.

    > Raciocínio análogo ao anterior vale para Strings e inteiros(criados com 'new Number(...)').

    > Comparar 2 objetos(x e y, digamos) x==y ou x===y sempre retornará falso.

    > Hoisting: posso declarar uma variável depois de usá-la, pois a declaração de variável em JS vai para o topo antes mesmo do código ser executado. MAS É APENAS A DECLARAÇÃO. NÃO EXISTE HOISTING DE INICIALIZAÇÃO EM JS.

    > Também existe Hoisting de Função em JS: seu nome e seu corpo serão hoisted.

    > JS só tem escopo a nível de função. Exemplo: se você criar uma variável dentro do if, ela terá aquele valor mesmo fora(if não define um escopo como em outras liguagens cuja definição de escopo é por bloco, como C/C++ e Java). Se quisermos simular a definição de escopo de outras linguagens, podemos usar clojures para isso.

    > Strict Mode: não podemos usar variáveis que não foram declaradas.

    > Nunca instancie tipos básicos com as classes(string, int, boolean etc) através de 'new'; instancie, no entanto, com valores simples. Ex: var array = []; var str = ""; var int = 0; etc

    > Atribua valores default para os parâmetros caso eles sejam setados para undefinied.

    > Evite usar eval()

    > Evite usar variáveis desnecessárias.

    > dica de performance: acesse o DOM apenas uma vez. Ex: obj = document.getElementById("demo"); pois esse getElementById é lento. Faça só uma vez esse get.

    > dica de performance: não faça array.length no for(a cada iteração eu teria que consultar esse método). Crie uma variável e atribua a ela: var l = array.length;

    > evite usar palavras reservadas de java, js e html como nome para variáveis.

    > Evite usar ECMAScript 6 ainda é parcialmente suportada pelos navegadores. Por enquanto, utilize o ECMAScript 5.

    > O formato textual JSON(javascript object notation) é sintaticamente idêntico à criação de objetos javascript. Isso facilita a conversão de JSON para JS nativo.
